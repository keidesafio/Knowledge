# FMEAエージェント開発プロジェクトに参画することになったので、AIエージェントを一から調べてみた

先日、FMEAの分析を自動化するAIエージェントの開発プロジェクトにアサインされました。  
「AIエージェント」という言葉は聞き慣れていたのに、いざ説明しようとすると意外と言語化できなくて、改めて調べ直しました。

同じように「なんとなく分かってるけど人に説明できない」という方向けに、整理してまとめます。

---

## まず結論だけ

> **AIエージェント = LLM（頭脳）＋ ツール（手足）＋ ループ（繰り返し実行）**

これだけです。ただ、それぞれが何を意味するのかを順番に押さえていきましょう。

---

## ChatGPTやClaudeとは何が違うの？

普段使っている ChatGPT や Claude は「チャットAI」あるいは「LLM（大規模言語モデル）」と呼ばれます。  
使い方のイメージはこうです。

```
ユーザーが質問する → AIが回答する → 終わり
```

めちゃくちゃ賢いのに、基本的に**呼ばれたら答えて、それで1回終わり**です。  
最近は Web 検索やコード実行もできますが、あくまで「ユーザーが操作して、AIが応答する」という構図は変わりません。

コンサルタントに例えると、**「質問するたびに呼ぶ、超優秀なスポットコンサルタント」**です。

---

## AIエージェントは何が違うのか

AIエージェントの動きはこうなります。

```
目標を受け取る
  ↓
情報を集める（Web検索、ファイル読み込みなど）
  ↓
行動する（コードを書く、ファイルを編集するなど）
  ↓
結果を確認する
  ↓
まだ終わってなければ → また考えて → また行動する
  ↓
完了
```

**「行動 → 結果確認 → また行動」を自分で繰り返す**のが最大の特徴です。  
ユーザーが逐一指示しなくても、目標を達成するまで自律的に動き続けます。

Anthropic（Claudeの開発元）はこう定義しています。

> **「エージェントとは、ループ内でツールを使うモデルである」**

コンサルタントの例えで言えば、**「プロジェクトに常駐して、自分で考えながら完了まで動き続けるプロジェクトマネージャー」**です。

---

## LLMとAIエージェントは「対立」ではなく「包含」関係

ここが一番混同しやすいポイントです。

**LLMはAIエージェントの「頭脳」部分です。**  
AIエージェントはその頭脳を内蔵したうえで、ツール・記憶・実行ループを加えたシステム全体を指します。

| | LLM（例：Claude、GPT） | AIエージェント（例：Claude Code） |
|---|---|---|
| 動き方 | 質問 → 回答（1往復） | 目標 → 計画 → 実行 → 確認 → 繰り返し |
| 自律性 | ユーザーが都度指示する | 自分で判断して動く |
| 作業範囲 | 基本的に1回の応答で完結 | 何十ステップでも継続して実行 |
| 例え | 超優秀なコンサルタント | 超優秀な常駐プロマネ |

「LLM vs AIエージェント、どっちがすごい？」という比較はそもそも成立しません。  
**AIエージェントの中でLLMが動いている**、というのが正しい関係です。

---

## ツールが「使える」と「自分で判断して使う」は別物

IBMはこう言っています。

> **「ツールを呼び出せるだけではエージェントにはならない。  
> LLMがどのツールをいつ使うか自分で判断できて初めてエージェントになる」**

最近のGPTやClaudeも Web 検索やコード実行のツールを持っています。  
ただ、「ユーザーに指示されてツールを使う」のと「目標達成のために自分でツールを選んで組み合わせる」のは、質が全く違います。

---

## 「頭脳が同じなのに、システムに入れると性能が上がる」という事実

具体的な数字があります。OpenAI の GPT を例にすると：

| 条件 | スコア |
|---|---|
| 素の GPT（ツールもループもなし） | 33.9% |
| Codex（フルスペックのエージェントシステム）に組み込んだ GPT | 51.3% |

**同じ頭脳を使っているのに、エージェントシステムに入れるだけで約1.5倍の性能になります。**

「モデルが賢ければそれで十分」ではなく、どういうシステムに組み込むかが性能を大きく左右する、ということです。  
Claude Code も同じ構造で、Claude（LLM）をエージェントシステムとして動かすことで、素のClaudeよりはるかに複雑な作業をこなせるようになっています。

---

## 今回のプロジェクト（FMEAエージェント）との関係

これを踏まえると、FMEAエージェントでやろうとしていることが整理できます。

- **頭脳（LLM）**：Claude など → 自然言語でFMEAの文脈を理解し、リスクを推論する
- **手足（ツール）**：ドキュメント読み込み、過去のFMEAデータ参照、結果の書き出しなど
- **ループ**：「情報が足りなければもう一度調べる」「記載が曖昧なら確認しにいく」を自律的に繰り返す

人が時間をかけて行う「資料を読む → リスクを洗い出す → 分析する → まとめる」という一連の作業を、ループとツールの組み合わせで自動化するのがこのプロジェクトの本質です。

---

## まとめ

- **LLM**：言語を理解・生成する頭脳。単体では「呼ばれたら答えるだけ」
- **AIエージェント**：LLM ＋ ツール ＋ ループ。目標を与えれば自律的に完了まで動く
- **ポイント**：ツールが「使える」かではなく、「自分で判断して使いこなせる」かがエージェントかどうかの境目
- **性能**：同じLLMでも、エージェントシステムに組み込むことで単体より大幅に性能が上がる

「賢いAIが返答してくれる」から「AIが自分で動いて仕事を完了させる」へ。  
この違いを掴んでおくと、プロジェクトの設計議論でも話についていきやすくなると思います。
